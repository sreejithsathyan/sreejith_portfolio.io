Event Driven Programming Task

Implement an event-based IoT node solution in this task. Modelling the IoT node as a finite state machine. The node has three states: Processing, Sensing, Idle. The node is moving from Idle to some event's sensing. As a clock, the node switches from sensing to processing once the data is collected and from processing to idle when the data is processed.

UML State Diagram

state diagram
In this task, there are three nodes, Sensing, Processing, Ideal. 

Sensing

A set of weather sensors temperature, humidity and wind, which collect data for 1 minute and stored into a data structure array, values is stimulated using random value generator in c++

Processing

Weather data stored in array is pass to processing function, which summation all the values independently and calculate the average of temperature, humidity, wind. the result includes time and location also is displayed on the console and write as a sensor log file  

Ideal

The ideal state is an intermediate state between processing to sensing which just delay for sometime after one iteration
